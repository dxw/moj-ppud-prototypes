Public Protection Unit Database

From May - June 2020 we did a discovery to explore what would be required for the Public Protection Unit Database (PPUD) to become a modern, flexible, supportable, and user-centred service that:

  • enables the effective and secure release and recall of offenders
  • diverts patients to the mental health system
  • supports the investigation of complaints and fatal incidents in justice settings

A number of significant issues were identified with the existing service that led to a recommendation that the monolithic PPUD service be deprecated and replaced by a number of modern services, centred around the outcomes that each organisation using PPUD is trying to achieve, built according to modern technology standards. These new services should be hosted, maintained and supported by the MoJâ€™s Digital Studio.

From July to October 2020 we run an Alpha with 3 objectives:

  • De-risk data migration
  • What information can be reused or repurposed
  • Allow for better collaboration and more efficient work

Our recommendation is to build different services with common components to better reflect the user journeys they support

  • Managing a parole review (PPCS-PB)
  • Managing a recall (PPCS-PB)
  • Managing a transfer to a hospital (MHCS)
  • Managing a complaint in a justice setting (PPO)
