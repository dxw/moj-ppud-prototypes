Manage a parole review

What we did

PPCS - Calculate tariff expiry date

Currently to calculate the Tariff expiration date some have their own spreadsheet with formulas, others do it manually. It doesn't need to be that way as some can be calculated automatically.

We tested a prototype to help PPCS to calculate it so the review goes ahead.

PPCS - Collect documents for the dossier

Currently PPCS collects and chases all the documents via email, that they need to then manually upload on PPUD.

We tested prototypes that will help them check and collect the right documentation.

MCA - Check documents for the Parole Board

Currently MCA has to manually check on PPUD the dossiers, with lots of back and forth emails with PPCS.

We tested some prototypes to simplify their workflow and help MCA caseworkers verify the documentation and refer cases, so the Parole Board has the right information

What we didn't do

  • Scheduling
  • Work allocation
  • Testing in real time how handovers would work

To explore further

  • When surfacing key information, most participants wanted to see every document. In some cases it made sense as it helped them understand the context of the case, but in others we couldn't see a clear reason. This has happened in most of the sessions and we wonder if it's a “habit”, rather than an actual need.
  • Test with more Community offender managers the email with a magic link to provide information and if it should be password protected or they'll need an account

What we learned

PPCS Initial Action team - Calculate tariff expiry date

  • The overview was useful to every participant, however some of them didn’t understand at the beginning that the information about the offender was automatically pulled from Nomis.
  • In the cases where we can’t provide the data automatically, having the relevant documents close to them saved users time.
  • Dividing their to-do list on new or in-progress cases wasn’t useful. The second iteration organised by groups of tasks worked well. (ie. Check and request documents, Calculate TED etc)

PPCS - Collect documents for the dossier

  • To onboard users, the new product should explain what’s different and what they don’t need to do anymore. Start pages worked well for this.
  • Documents automatically checked for errors weren’t always trusted. Some users preferred to check it themselves.
  • Being able to request documents and send automatic reminders was really useful and time saving for users.

Community Offender Manager – Provide documents

Because of research participants availability we weren't able to conduct any concept testing with participants.

MCA - Check documents for the Parole Board

  • Showing the dossier overview and structure worked. Participants recognised the documents that needed additional checks.
  • Similar to previous research sessions, documents automatically checked for errors weren’t always trusted. Some users preferred to check it themselves.
