Technical Architecture diagrams

05 October 2020

In order to describe the future system architecture, and perform analysis of the legacy PPUD database, we created a series of diagrams:

  • Overall system context diagram
  • Container diagrams for new frontend and API services
  • Legacy database entity-relationship diagrams
  • High-level service-oriented ERDs for the legacy database

The linked repository contains both rendered diagrams, and PlantUML source code for them.

PPUD Legacy API wrapper

18 September 2020

We decided to build out a prototype for wrapping the existing MSSQL database with a modern API using a small subset of PPUD shaped data. It will enable is to expose data to authenticated clients, such as any new PPUD replacement services.

The API wrapper proves that we can:

  • Expose structured data from the existing PPUD database
  • Expose related documents via a data access layer
  • Use open standards

PDF validation

18 August 2020

We identified an issue where users will accidentally upload incorrect documents on cases.

It is possible to validate PDFs based on their content. Using OCR we can read the text whether it is a scanned document or not.

This experiment only uses simple string matching, matching names can be complicated however as long as we don't prevent users from uploading documents, just advise them to double check where we think text is missing that should go someway to reducing erroneous documents being uploaded.

API Mocks

5 August 2020

We identified that access to test environment and APIs may become a blocker to deveopment teams.

In order to remove that blocker, we tested whether it was possible to provide mocked APIs and configure our own test data, using published API specifications.

This application provides mock APIs for the RESTful Prison and Community APIs, as well as the SOAP-style PPUD WAM API, should it be needed.
